# Blog Content Plan — 16 New Posts from the 100-Question Knowledge Base

Built from [maysa-qa-100-review.md](../maysa-qa-100-review.md). Each of the 100 guest
questions is mapped to a blog post so the same answer serves two jobs at once:
**SEO** (an indexable page targeting the real search phrase) and **bot knowledge**
(the chatbot can cite the article for depth, and its published title/URL is already
fed into the system prompt).

## How this was scoped
- The existing 25 posts (5 pillars × 5) already cover a chunk of the 100 questions —
  those are **not** duplicated here; instead the new posts **link to** them.
- The 16 new posts below fill the genuine gaps and cluster related questions so each
  page has real depth (a page answering one thin question ranks poorly and reads
  like filler).
- Every internal link uses a **real slug** — existing slugs verified against
  `blog-content/pillar-*.md`, new slugs listed in the "New slugs" table so cross-links
  between the new posts resolve once published.
- Meta title ≈ 55–60 chars; meta description ≈ 150–160 chars (both within Google's
  typical display limits).
